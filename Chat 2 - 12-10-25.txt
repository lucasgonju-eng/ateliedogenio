Você pode abrir o front-end com o “Go Live” do VS Code, mas ele é bem diferente do servidor PHP nativo e, neste projeto, cada um atende a um pedaço específico:

PHP integrado (php -S localhost:8080 -t public)

Serve os arquivos PHP, lê index.php, passa pelas rotas e executa o backend (controllers, middlewares, Supabase etc).
Sem ele, nada que dependa de bootstrap/app.php, rotas, CSRF, autenticação e chamadas para o Supabase funciona, porque o Go Live só entrega HTML/JS, não roda PHP.
Mantém as sessões ($_SESSION, cookies), gera tokens CSRF e responde a /sales, /auth/login, /products/options… Ou seja, é o servidor “real” da aplicação.
Go Live (Live Server do VS Code)

É um servidor estático: só entrega o arquivo tal qual está no disco.
Útil para protótipos em HTML/JS puro ou para ver rapidamente um layout, mas ele nunca executa PHP (o browser veria os <?php ... ?> como texto).
As chamadas AJAX que fizemos com window.apiFetch('/sales', …) continuariam apontando para localhost:PORT/sales. Se esse endpoint não estiver rodando no servidor PHP, a requisição falha.
